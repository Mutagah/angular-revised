Life Cycle Hooks

For results in this section, please check the console

ngOnInit

ngOnInit is invoked when the component has been initialized. It is called only once after the first ngOnChanges

Mostly is used for doing initializations and once a component is loaded into the DOM, the components constructor and the onInit method are called.

Constructors are called first

ngOnDestroy

ngOnDestroy will be called once just before component is removed from the DOM.

If we need any cleaning activities we can do it on onDestroy

ngOnChanges

onChanges will be called once before onInit and will be called whenever the data-bound input properties changes.

Data-bound input is not the normal input in the same component rather it is something coming from outside of that component through @Input.

ngOnChanges can used to do something if a particular value changes

ngDoCheck

DoCheck will be invoked when the change detector of the given component is invoked.

It is called immediately after ngOnChanges on every change detection and immediately after onInit on the first run.

ngOnInit, ngOnDestroy, ngOnChanges and ngDoCheck are general life cycle hooks both present in components and directives.

NB: Avoid using ngOnChanges and ngDoCheck in the same componenent

ngAfterContentInit

ngAfterContentInit will be called only once after the first doCheck

Content projection : This is when you project some contnet into child from parent. Normally the content is wrapped with the child tags then using ng-content directive in the child component html file, you specify where you want the content to appear.

Content projection is very useful when we are creating reusable widgets.

ngAfterContentChecked

This is called after ngAfterContentInit and every subsequent ngDoCheck

ngAfterViewInit

ngAfterViewInit will be called once after the first ngAfterContentChecked.

ngAfterViewChecked

ngAfterViewChecked will be called once after ngAfterViewInit and after every subsequent ngAfterContentChecked

The static in View Child Decorator

The second parameter on ViewChild or ContentChild which is an object with the words static: true makes content accessibe in the ngOnChanges and ngOnInit itself
